# Дизайн‑док — "Последний рейс" (Mobile‑first)

Версия: 1.0
Автор: ниш3вый / cfmz3

---

1. Жанр и платформа

- Жанр: Психологический хоррор / симулятор ходьбы от первого лица с элементами квеста и морального выбора.
- Платформа: Android (первичная), ПК (опционально позже).
- Движок: Godot 4.3.
- Камера: От первого лица.
- Управление: виртуальные элементы для мобильных (joystick, кнопка взаимодействия), поддержка клавиатуры и мыши для теста на ПК.

2. Сеттинг и атмосфера

- Время: конец 1980-х, вымышленная восточноевропейская страна с тоталитарным режимом.
- Основное место: ночной пассажирский поезд дальнего следования (советского типа).
- Визуал: проработанные PBR‑текстуры (грязь, царапины, потертости), детализированные материалы, постобработка для мобильных (тональная коррекция, vignetting).
- Атмосфера: сырость, запах мазута, тусклое мигающее освещение, шёпоты, стук колёс.

3. Сюжет и структура

(См. исходный документ — пролог, 4 акта, ветвления и концовки). Игра линейная по структуре актов, с развилками, приводящими к 4 главным концовкам.

4. Персонажи

- Машинист (игрок)
- Грета, Эрих, Шварц, Бруно, Лена, Йозеф, Марта, Феликс, Карл, Райнер.

5. Квесты и моральные дилеммы

Основные квесты и выборы указаны в исходном документе. Для мобильной версии важно, чтобы квесты были короткими и давали понятные последствия.

6. Визуальный стиль и оптимизация для мобильных

- Стиль: реалистичный, проработанные текстуры и материалы (не low‑poly).
- Текстуры: целевой главный размер 2048 для ключевых персонажей/интерьеров на high-end Android (рекомендуется для первой итерации использовать 1024 для большинства объектов и 2048 только для лиц/близких объектов). Для экономии: base albedo 1024/normal 1024, AO/roughness объединены в комбинированные карты при необходимости.
- Полигоны (mobile targets):
  - Персонаж (PC): 30k–40k, Mobile: 8k–15k (LOD0 15k, LOD1 6k, LOD2 2–3k)
  - Купе / секция вагона: Mobile LOD0 12k–20k, LOD1 5k–8k, LOD2 2–3k
  - Пропсы малые: 500–3k в зависимости от важности
- Формат текстур для Android: ETC2 (RGBA) или ASTC для новых устройств; экспорт PNG исходников и конвертация при билде.
- Экономия: атласирование мелких пропсов, инстансинг деревьев и полосы рельс как тайл-меш.

7. Технические требования для ассетов

- Формат: .glb (glTF 2.0). Y‑up, метры.
- Текстуры: PNG исходники (1024/2048), в проекте оптимизировать/конвертировать.
- Скелет и анимации: экспортировать внутри .glb для персонажей.
- Коллайдеры: простые примитивы или экспортированные объекты с именем *_collider.
- LOD: обязательно для ключевых моделей.

8. Звук и музыка

- Формат: .ogg для looping музыки, .wav для SFX исходников.
- Музыка: ambient loops 60–180 сек.
- Эффекты: стук колёс, шёпоты, лампы, двери.

9. Диалоги и локализация

- Версия 0: RU (полные тексты диалогов) — позже добавить EN.

10. Pipeline генерации ассетов (high→bake→game)

1. High poly sculpt (ZBrush/Blender) → 2. Retopo → 3. UV and bake (normal/AO) → 4. Texturing (Substance/Quixel) → 5. Export game mesh (target tris) + baked maps → 6. LOD generation → 7. Export .glb
