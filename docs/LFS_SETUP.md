# Git LFS и хранение ассетов — инструкция

Мы рекомендуем хранить тяжёлые ассеты (.glb, текстуры, звуки) через Git LFS или внешнее хранилище (S3/Google Drive). Для мобильной разработки особенно важно контролировать размер APK / AAB и использовать компрессию текстур (ASTC/ETC2).

Рекомендованная настройка (локально):
1. Установите Git LFS: https://git-lfs.github.com/
2. Инициализируйте в репозитории:
   git lfs install
3. Убедитесь, что .gitattributes уже содержит правила (файл в корне репо).
4. Добавляйте большие файлы и закоммитьте:
   git add path/to/asset.glb
   git commit -m "Add asset...")

Если вы предпочитаете внешнее хранилище:
- Храните исходники (high-poly, PSD, ZBrush) в защищённом хранилище (S3/Google Drive).
- В репозитории храните только промпты, манифесты и lightweight preview .glb (LOD2).
- Добавляйте в meta/assets_manifest.json ссылку на хранилище для каждого heavy asset.

Примечание по текстурам:
- Исходники PNG 2048/1024 сохраняйте отдельно и конвертируйте в ASTC/ETC2 в CI или во время билда.
